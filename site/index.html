What it does

One job. Every page, every state, captured.

SunSponge crawls a site or expands a map you already have, then drives Playwright to screenshot each page at every viewport and color scheme. The output is clean PNGs, named so you can tell exactly what each one is.

Discover

Crawl it, or hand it a map

Let SunSponge crawl from a URL, read an XML sitemap, or take a pathway map — a Markdown manifest or a rhobear-verifier JSON report — and skip blind crawling entirely.

Viewports

Desktop, tablet, phone

Every page captured at each width, so the responsive states are in the box too.

Schemes

Light and dark

Both color schemes per page, driven through the real browser.

Named output

You can read the filenames

001-capture-start-wired-desktop-light.png — id, status, viewport, scheme. No mystery files.

Download

One ZIP, yours

Start a job, poll it, pull the ZIP. Local service, your machine.

No AI — on purpose

Just Playwright, a crawl plan, and PNG output.

There's no model in the loop, no key to bring, nothing phoning home. SunSponge is a deterministic capture engine: feed it the same site and map, get the same shots. That's what makes it dependable for audits, design reviews, and before/after diffs.

How it works

Four steps. No model, no account.

01

Point

Give it a base URL, an XML sitemap, or a pathway map you already have.

02

Plan

It builds the crawl plan and expands every pathway into capture targets.

03

Capture

Playwright shoots each page × viewport × scheme to clean PNGs.

04

Download

Pull a named ZIP. Diff it, archive it, drop it in a review. You own it.
